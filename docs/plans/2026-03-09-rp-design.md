# RP: Roleplay Chat — Design Document

**Date:** 2026-03-09
**Status:** Approved
**Depends on:** projects/aiserver, projects/db

## Overview

A roleplay chat application built as an aiserver plugin. Supports SillyTavern v2 character cards, scenario cards, variable expansion, streaming chat, and a pre/post processing pipeline for prompt assembly and response handling.

## Architecture

RP is an **aiserver plugin**. Aiserver gets a minimal plugin system: each plugin is a Python package that exports `register(app, ollama)`. Plugins are discovered via `config.json`:

```json
{
  "plugins": [
    {"name": "rp", "path": "../rp"}
  ]
}
```

On startup, aiserver adds each plugin's path to `sys.path`, imports it, and calls `register()`. The plugin receives the FastAPI app (to mount routes and static files) and the shared `OllamaClient` instance.

## Processing Pipeline

A chain of hooks that run before and after each LLM call. Each hook receives a context dict and returns it (modified or not).

### Pre-processing chain

1. **Variable expansion** — replaces `${user}`, `${char}`, `${scenario}` etc. with values from loaded cards
2. **Prompt assembly** — builds system prompt (scenario + char description + personality) + chat history
3. **Context management** — applies active strategy to fit within token limit

Produces `prompt` and `system` strings sent to OllamaClient.

### Post-processing chain

1. **Response cleaning** — strip artifacts, fix formatting
2. *(Future: mood detection, action parsing, etc.)*

```python
class Pipeline:
    def __init__(self):
        self.pre_hooks: list[Callable] = []
        self.post_hooks: list[Callable] = []

    async def run_pre(self, ctx: dict) -> dict: ...
    async def run_post(self, ctx: dict) -> dict: ...
```

The `ctx` dict carries: cards, scenario, history, prompt, response, model config.

## Context Management

Strategy interface for swappable context fitting:

```python
class ContextStrategy:
    def fit(self, messages: list[dict], max_tokens: int) -> list[dict]:
        """Return messages that fit within the token budget."""
        ...
```

Ships with **SlidingWindow** — drops oldest messages first, always keeps system prompt and first message (character greeting). Active strategy set per-scenario in `settings` JSONB.

Future strategies: summarization, RAG retrieval, importance scoring.

## Prompt Template

Fixed structure for v1 (plan for configurable templates later):

```
[System] scenario_description + char_description + char_personality
[History] first_message + conversation messages (fitted by context strategy)
[User] latest user message
```

## Chat UI

Single-page web app at `/rp/` with three views:

### Chat view
- Message bubbles (user left, AI right) with character avatars
- Streaming tokens (NDJSON reader)
- Thinking tokens in collapsible section
- Under-the-hood panel (collapsible): assembled prompt, token counts, timing stats
- Model selector, regenerate button, edit/delete messages

### Cards view
- Card list with avatar thumbnails
- Import SillyTavern PNG (drag & drop)
- Card editor: name, description, personality, first message, example messages, scenario, tags
- Export as SillyTavern PNG

### Scenarios view
- Scenario list
- Editor: name, description (variable reference), model override, context strategy picker
- "New Chat" button: pick user card + AI card + scenario → starts conversation

## API Routes


## SillyTavern Card Format

Uses SillyTavern Character Card v2 spec. Cards are PNG images with JSON metadata embedded in the `tEXt` chunk under the key `chara` (base64-encoded). Key fields:

- `name`, `description`, `personality`, `first_mes`, `mes_example`
- `scenario`, `creator_notes`, `system_prompt`, `post_history_instructions`
- `tags`, `creator`, `character_version`
- `extensions` (JSONB for custom data)

Import reads the PNG tEXt chunk, export writes it back.
